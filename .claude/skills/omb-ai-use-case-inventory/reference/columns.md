# OMB Inventory Column Reference

Per-field detail for all 36 OMB-defined columns plus the 2 OMB-added consolidation columns.

For each column: original Excel header (verbatim â€” including embedded line breaks), data type, when it's required, canonical valid values, and the shorthand recoding code we use for compact display. The `recoding_map` tables in `recoding_maps.md` document the messy aliases seen in the wild that fold to each canonical value.

## Section 0 â€” OMB-added columns (not filed by agencies)

### 1. `agency`
- **Source:** generated by OMB during consolidation
- **Data type:** free_text
- **Meaning:** Abbreviated agency name (e.g., `USDA`, `DHS`, `HHS`)

### 2. `agency_name`
- **Source:** generated by OMB
- **Data type:** free_text
- **Meaning:** Full agency name (e.g., `Department of Agriculture`, `Department of Health and Human Services`)

## Section 1 â€” Identifiers

### 3. `id`
- **Excel header:** `Use Case ID  [Agency Abbrev.] â€“ [#]`
- **Data type:** structured_text
- **Required:** Yes
- **Meaning:** Unique identifier for the AI use case. Should remain stable year-over-year if the use case persists. Agencies are permitted to omit this column from public-facing inventories.
- **DB column:** `use_cases.use_case_id` â€” Intentional divergence â€” kept the legacy DB name because renaming to `id` would conflict with the table's primary key.

### 4. `use_case_name`
- **Excel header:** `Use Case Name`
- **Data type:** free_text
- **Required:** Yes
- **DB column:** `use_cases.use_case_name`

### 5. `agency_bureau`
- **Excel header:** `Bureau/Component`
- **Data type:** free_text
- **Required:** Yes
- **Meaning:** Specific organization(s) within the agency responsible for the AI use case. **The most variable column in practice** â€” see SKILL.md "Where the messy data hides" for parsing patterns by agency.
- **DB column:** `use_cases.bureau_component` â€” Intentional divergence â€” kept the legacy DB name due to call-site density.

### 6. `contact_email`
- **Excel header:** `Email Address`
- **Data type:** free_text
- **Required:** Yes
- **Meaning:** Email of the responsible team. Public inventories may substitute a designated public-inquiries email.
- **DB column:** `use_cases.email_address`

## Section 2 â€” Filing posture

### 7. `is_withheld`
- **Excel header:** `Should this AI use case be withheld from public reporting?`
- **Data type:** multiple_choice
- **Required:** Yes
- **Valid selections:**
  - `a) No`
  - `b) Yes â€“ agency has determined that there's a risk to disclosure, such as a harm to an interest protected by a FOIA exemption`
  - `c) Yes â€“ disclosure is prohibited by law`
  - `d) Other`
- **Shorthand:** `No` / `Yes - Disclosure Risk` / `Yes - Prohibited by Law` / `Other`
- **DB column:** `use_cases.is_withheld`

### 8. `development_stage`
- **Excel header:** `Stage of Development`
- **Data type:** multiple_choice
- **Required:** Yes
- **Valid selections:**
  - `a) Pre-deployment â€“ The use case is in a development or acquisition status.`
  - `b) Pilot â€“ The use case has been deployed in a limited test or pilot capacity.`
  - `c) Deployed â€“ The use case is being actively authorized or utilized to support the functions or mission of an agency.`
  - `d) Retired â€“ The use case was reported in the agency's prior year's inventory, but its development and/or use has since been discontinued.`
- **Shorthand:** `Pre-deployment` / `Pilot` / `Deployed` / `Retired`
- **DB column:** `use_cases.stage_of_development` â€” Intentional divergence â€” kept the legacy DB name due to call-site density (49 refs across dashboard CASE-statement normalization + audit docs).

### 9. `is_high_impact`
- **Excel header:** `Is the AI use case high-impact?`
- **Data type:** multiple_choice
- **Required:** Yes
- **Valid selections:**
  - `a) High-impact`
  - `b) Presumed high-impact, but determined not high impact`
  - `c) Not high-impact`
- **Shorthand:** `High-impact` / `Presumed High-Impact, but Not High-impact` / `Not High-impact`
- **Reference:** Definition is in Section 5 of the appendix to OMB Memorandum M-25-21.
- **DB column:** `use_cases.is_high_impact`

### 10. `HI_justification`
- **Excel header:** `Justification`
- **Data type:** free_text
- **Required:** Conditional â€” only when `is_high_impact == "b) Presumed high-impact, but determined not high impact"`
- **DB column:** `use_cases.justification`

## Section 3 â€” What the AI does

### 11. `topic_area`
- **Excel header:** `Use Case Topic Area`
- **Data type:** multiple_choice
- **Required:** Yes for pre-deployment / pilot / deployed (not retired)
- **Valid selections:** Administrative Functions, Cybersecurity, Emergency Management, Energy and the Environment, Government Benefits Processing, Health and Medical, Human Resources, Information Technology, International Affairs, Law Enforcement, Other, Procurement and Financial Management, Science, Service Delivery, Transportation
- **Shorthand examples:** `Admin Functions`, `Cybersecurity`, `Emergency Mgmt`, `Energy and Environment`, `Benefits Processing`, `Health and Medical`, `HR`, `IT`, `Procurement and Finance Mgmt`
- **DB column:** `use_cases.topic_area`
- **Note:** Agencies often file finer-grained topic names ("Internal Knowledge Management & Staff Support", "Software Development & Engineering", "Communications & Public Affairs"). These get recoded to the OMB canonical buckets per `recoding_maps.md`.

### 12. `classification`
- **Excel header:** `AI Classification`
- **Data type:** multiple_choice
- **Required:** Yes for pre-deployment / pilot / deployed
- **Valid selections (full text):**
  - `Agentic AI: AI systems that perform tasks or make decisions autonomously with minimal human intervention.`
  - `Classical/Predictive Machine Learning: Models trained on data to make predictions or classifications based on identified patterns or relationships.`
  - `Computer Vision: AI that processes and interprets visual data (e.g., images and videos).`
  - `Generative AI: AI that generates new or synthetic content (e.g., images, videos, audio, text, code).`
  - `Natural Language Processing: AI that processes, interprets, and shares information in human language.`
  - `Reinforcement Learning: AI trained through trial and error using rewards and penalties to optimize decision-making policies.`
- **Shorthand:** `Agentic AI` / `Classical ML` / `Computer Vision` / `Generative AI` / `NLP` / `Reinforcement Learning`
- **DB column:** `use_cases.ai_classification`
- **Note:** Use case may rely on multiple classifications; agency selects the closest single match.

### 13. `problem_solved`
- **Excel header:** `What problem is the AI intended to solve?`
- **Data type:** free_text
- **Required:** Yes for pre-deployment / pilot / deployed
- **DB column:** `use_cases.problem_statement`

### 14. `benefits`
- **Excel header:** `What are the expected benefits and positive outcomes from the AI for an agency's mission and/or the general public?`
- **Data type:** free_text
- **Required:** Yes for pre-deployment / pilot / deployed
- **DB column:** `use_cases.expected_benefits`

### 15. `system_outputs`
- **Excel header:** `Describe the AI system's outputs.`
- **Data type:** free_text
- **Required:** Yes for pre-deployment / pilot / deployed
- **Meaning:** Whether the output is a prediction, recommendation, decision, generated artifact, etc.
- **DB column:** `use_cases.system_outputs`

### 16. `operational_date`
- **Excel header:** `Date when AI use case became operational or the pilot's start date`
- **Data type:** date
- **Required:** Yes for pilot / deployed (not pre-deployment)
- **DB column:** `use_cases.operational_date`

## Section 4 â€” How it was built

### 17. `contracting_usage`
- **Excel header:** `Was the system involved in this use case purchased from a vendor or developed under contract(s) or in-house?`
- **Data type:** multiple_choice
- **Required:** Yes for pilot / deployed
- **Valid selections:**
  - `a) Purchased from a vendor`
  - `b) Developed in-house`
  - `c) Developed with both contracting and in-house resources`
- **Shorthand:** `Vendor Purchased` / `In-house Development` / `Contracting and In House`
- **DB column:** `use_cases.development_type` â€” Intentional divergence â€” kept the legacy DB name due to call-site density (66 refs).

### 18. `vendor_name`
- **Excel header:** `Vendor(s) Name`
- **Data type:** free_text
- **Required:** Conditional â€” yes when `contracting_usage` is "Purchased" or "Both", AND deployment stage is pilot or deployed
- **DB column:** `use_cases.vendor_name`
- **Note:** Often a list of contractors / system integrators rather than the actual product vendor.

### 19. `have_ato`
- **Excel header:** `Does this AI use case have an associated Authorization to Operate (ATO)?`
- **Data type:** multiple_choice
- **Required:** Yes for pilot / deployed
- **Valid selections:** `Yes` / `No`
- **Common aliases:** `true`/`false`, `Use vendor's ATO or FedRAMP authorization`, `In-progress - ATO process underway` (treat as `Yes`)
- **DB column:** `use_cases.has_ato` â€” Intentional divergence â€” kept the legacy DB name due to call-site density (33 refs incl. dashboard ATO availability matrix).

### 20. `system_name_ato`
- **Excel header:** `System(s) Name`
- **Data type:** free_text
- **Required:** Conditional â€” yes when `have_ato == "Yes"` AND deployment stage is pilot or deployed
- **DB column:** `use_cases.system_name` â€” Intentional divergence â€” kept the legacy DB name due to call-site density (82 refs, large audit-doc surface area).

### 21. `data_description`
- **Excel header:** `Describe any data used to train, fine-tune, and/or evaluate performance of the model(s) used in this use case.`
- **Data type:** free_text
- **Required:** Yes for pilot / deployed
- **Meaning:** Datasets used (research / public / external / agency-internal), plus vendor-supplied descriptive info where applicable.
- **DB column:** `use_cases.training_data_description` â€” Intentional divergence â€” kept the legacy DB name due to call-site density (37 refs incl. test fixtures).

### 22. `link_to_data`
- **Excel header:** `If the data is required to be publicly disclosed as an open government data asset, provide a link to the entry on the Federal Data Catalog.`
- **Data type:** free_text
- **Required:** No
- **DB column:** `use_cases.link_to_data`

### 23. `has_pii`
- **Excel header:** `Does this AI use case involve personally identifiable information (PII) that is maintained by the agency?`
- **Data type:** multiple_choice
- **Required:** Yes for pilot / deployed
- **Valid selections:** `Yes` / `No`
- **Common aliases:** `b) Yes`, `a) No`, lowercased variants, agency-narrative substitutions like `Yes - Customer contact and account information` (treat as `Yes`)
- **Reference:** PII as defined in OMB Circular A-130
- **DB column:** `use_cases.has_pii`

### 24. `pia_url`
- **Excel header:** `If publicly available, provide the link to the AI use case's associated Privacy Impact Assessment (PIA).`
- **Data type:** free_text
- **Required:** No
- **DB column:** `use_cases.pia_url`

### 25. `demographic_features`
- **Excel header:** `Which, if any, demographic variables does the AI use case explicitly use as model features?`
- **Data type:** select_multiple
- **Required:** Yes for pilot / deployed
- **Valid selections:**
  - `a) Race/Ethnicity`
  - `b) Sex`
  - `c) Age`
  - `d) Religious Affiliation`
  - `e) Socioeconomic Status`
  - `f) Ability Status`
  - `g) Residency Status`
  - `h) Marital Status`
  - `i) Income`
  - `j) Employment Status`
  - `k) None of the above`
  - `l) Other`
- **Storage:** stringified Python list (e.g., `['a) Race/Ethnicity', 'c) Age']`)
- **DB column:** `use_cases.demographic_features`

### 26. `has_custom_code`
- **Excel header:** `Does this project include custom-developed code?`
- **Data type:** multiple_choice
- **Required:** Yes for pilot / deployed
- **Valid selections:** `Yes` / `No`
- **Reference:** "Custom developed code" as defined in OMB Memorandum M-25-21
- **DB column:** `use_cases.has_custom_code`

### 27. `code_url`
- **Excel header:** `If the code is open source, provide the link for the publicly available source code.`
- **Data type:** free_text
- **Required:** No
- **DB column:** `use_cases.code_url`

- **Excel header:** `What are the potential impacts of using the AI for this particular use case and how were they identified?` (Subpractice: Complete AI Impact Assessment)

- **DB column:** `use_cases.hi_ongoing_monitoring`

### 33. `hi_training_established`
